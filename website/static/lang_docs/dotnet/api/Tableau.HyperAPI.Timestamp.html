Class Timestamp

Hyper TIMESTAMP and TIMESTAMP_TZ values - combination of date and time of day.

public sealed class Timestamp : ValueType, IComparable, IComparable<Timestamp>, IEquatable<Timestamp>
Remarks

Timestamp is almost identical to DateTime, except for two things:

  • Timestamp supports a greater range of dates.
  • DateTime resolution is .NET tick, which equals to 100 nanoseconds, while Timestamp resolution is 1 microsecond (10 ticks).

The Kind property indicates if a Timestamp represents UTC, local time, or is unspecified. However, a timestamp does not store an explicit time zone offset. In a time-zone-aware application, you must rely on some external mechanism to determine the time zone in which a Timestamp object was created.

Remarks

You can explicitly set the Kind property of a new Timestamp value to a particular System.DateTimeKind value by calling the SpecifyKind(Timestamp, DateTimeKind) method.

The Kind property allows a Timestamp value to clearly reflect either Coordinated Universal Time (UTC) or the local time. In contrast, the DateTimeOffset structure can unambiguously reflect any time in any time zone as a single point in time.

Creates a new Timestamp object that has the same date and time as the specified Timestamp, but is designated as either local time, Coordinated Universal Time (UTC), or neither, as indicated by the specified System.DateTimeKind value.

System.DateTimeKind kind

Returns
Type Description
Timestamp

A new object that has the same date and time as the object represented by the value parameter and the System.DateTimeKind value specified by the kind parameter.
Remarks

A Timestamp object consists of a Kind field that indicates whether the time value is based on local time, Coordinated Universal Time (UTC), or neither, and a date time components. The SpecifyKind(Timestamp, DateTimeKind) method creates a new Timestamp object using the specified kind parameter and the original date time components.

The returned Timestamp value may not represent the same instant in time as the value parameter, and SpecifyKind(Timestamp, DateTimeKind) is not a time zone conversion method. Instead, it leaves the date time components specified by the value parameter unchanged, and sets the Kind property to kind.

The SpecifyKind(Timestamp, DateTimeKind) method is useful in interoperability scenarios where you receive a Timestamp object with an unspecified Kind field, but you can determine by independent means that the datetime components represent local time or UTC.

Type Description
Timestamp

An object whose Kind property is System.DateTimeKind.Local, and whose value is the local time equivalent to the value of the current Timestamp object.
Remarks

If the Kind property is set to System.DateTimeKind.Utc, the current Timestamp object is converted to local time. If the Kind property is set to System.DateTimeKind.Unspecified the current Timestamp object is assumed to be a UTC time, and the conversion is performed as if Kind were System.DateTimeKind.Utc. No conversion is applied if the Kind property is set to System.DateTimeKind.Local.

Timestamp timestamp

Timestamp value to convert.

Type Description
System.DateTimeOffset
Remarks

Timestamp supports a greater range of dates, and the conversion throws an exception if the date is not representable by DateTimeOffset.

The offset of the resulting DateTimeOffset object depends on the value of the Kind property of the timestamp parameter:

If the value of the Kind property is System.DateTimeKind.Utc, the date and time of the DateTimeOffset object is set equal to timestamp, and its Offset property is set equal to 0.

If the value of the Kind property is System.DateTimeKind.Local or System.DateTimeKind.Unspecified, the date and time of the DateTimeOffset object is set equal to timestamp, and its Offset property is set equal to the offset of the local system's current time zone.
